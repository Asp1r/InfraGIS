# ROAD GIS Phase 1: эксплуатационный регламент

Документ описывает, как работать с дорожной моделью этапа 1 в InfraGIS: создавать дорогу, загружать ось, получать километраж и вести первичные записи IRI/дефектов.

## 1) Цель этапа

На этапе 1 система переводится в дорожную структуру:

- `Дорога` (`kind=road`) — контейнер проекта/участка.
- `Ось дороги` (`kind=road_axis`) — линейный референс.
- `IRI` (`kind=iri`) — слой продольной ровности.
- `Дефекты` (`kind=defects`) — слой дефектов по нормативам.

Ключевой принцип: диагностические данные привязываются к оси через километраж (`km_start`, `km_end`).

## 2) Модель данных

### 2.1 Слои (`layers`)

Основные поля:

- `id` — идентификатор слоя.
- `name`, `description` — метаданные.
- `kind` — тип дорожного слоя (`road`, `road_axis`, `iri`, `defects`).
- `source_type` — источник геометрии (`uploaded_geojson`, `url_geojson`).
- `parent_id` — родитель в дереве (для вложенных слоев).
- `axis_layer_id` — ссылка на ось (для `iri`/`defects`).

### 2.2 IRI (`iri_measurements`)

- `layer_id` — слой типа `iri`.
- `axis_layer_id` — ось дороги.
- `km_start`, `km_end` — линейная привязка участка.
- `iri_value` — значение IRI.
- `direction`, `lane` — направление/полоса (опционально).

### 2.3 Дефекты (`defect_records`)

- `layer_id` — слой типа `defects`.
- `axis_layer_id` — ось дороги.
- `defect_code` — тип дефекта.
- `severity` — уровень тяжести.
- `norm_ref` — нормативная ссылка (например, ОДМ/ГОСТ).
- `km_start`, `km_end` — линейная привязка.
- `extent_m`, `area_m2` — размеры дефекта.

### 2.4 Медиа 360 (`media_geo_links`)

- `media_id` — запись 360-медиа.
- `layer_id` — опциональный тематический слой/контекст.
- `axis_layer_id` — ось дороги, по которой выполняется линейная привязка.
- `lon`, `lat` — координаты точки съемки.
- `axis_km` — рассчитанный километраж точки по оси.

## 3) API этапа 1

### 3.1 Дерево дорожных слоёв

- `GET /layers/tree` — получить иерархию слоев.
- `POST /layers/roads` — создать слой дороги (`kind=road`).
- `POST /layers` — создать слой (в т.ч. `iri`/`defects`/`road_axis` в общем потоке).

### 3.2 Ось дороги и километраж

- `POST /layers/{road_id}/axis`
  - вход: файл или `source_url`;
  - поддержка форматов: GeoJSON, CSV, ZIP Shapefile;
  - выход: созданный слой оси + `total_km` + `points[]` с накопленным `km`.

### 3.3 Диагностические записи

- `POST /layers/iri-records`, `GET /layers/iri-records/{layer_id}`.
- `POST /layers/defect-records`, `GET /layers/defect-records/{layer_id}`.

### 3.4 Модуль 360 и линейная привязка

- `POST /media360/items/{media_id}/geolink`
  - вход: `lon`, `lat`, опционально `layer_id`, `axis_layer_id`;
  - выход: geolink с рассчитанным `axis_km` (если определена ось).
- `GET /media360/map-points`
  - возвращает точки 360 вместе с `axis_km`;
  - поддерживает фильтрацию по `axis_layer_id`.
- `POST /media360/axis/{axis_layer_id}/recalculate-km` (admin)
  - массово пересчитывает `axis_km` у существующих geolink;
  - опционально `media_id` для точечного пересчета.

## 4) Поддерживаемые форматы оси

### 4.1 GeoJSON

Поддерживается:

- `LineString`
- `MultiLineString`
- `Feature` (если внутри линейная геометрия)
- `FeatureCollection` (берутся только линейные геометрии)

### 4.2 CSV

Требования:

- обязательные координаты: `lon`/`lat` (или синонимы `lng|x`, `latitude|y`);
- минимум 2 точки;
- опциональный порядок: `order|seq|point_order|index`.

### 4.3 ZIP Shapefile

Архив обязан содержать:

- `.shp`
- `.shx`
- `.dbf`

Поддерживаются только полилинейные геометрии (`POLYLINE*`).

## 5) Операционный сценарий (администратор)

### Шаг 1. Создать дорогу

1. Открыть страницу администрирования слоев.
2. Нажать `Создать дорогу`.
3. Проверить, что в дереве появился узел `road`.

### Шаг 2. Загрузить ось дороги

1. Выбрать дорогу.
2. Загрузить ось файлом или URL.
3. Проверить ответ:
   - `total_km > 0`;
   - количество `points` соответствует ожидаемой геометрии.

### Шаг 3. Проверить дочерние слои

После загрузки оси должны быть доступны/созданы тематические слои:

- `IRI`
- `Дефекты`

### Шаг 4. Загрузить/создать диагностические записи

1. Создавать записи IRI в `iri_measurements`.
2. Создавать записи дефектов в `defect_records`.
3. Контроль:
   - `km_end >= km_start`;
   - корректный `axis_layer_id` (обязательно слой `road_axis`).

## 6) Проверки качества данных (чек-лист)

- [ ] Ось не содержит разрывов и дублирующихся «петляющих» сегментов.
- [ ] Начало оси согласовано с принятой инженерной направленностью трассы.
- [ ] Километраж монотонно возрастает по точкам оси.
- [ ] IRI и дефекты не выходят за диапазон фактической длины оси.
- [ ] У дефектов заполнены `defect_code` и `norm_ref`.
- [ ] Для всех диагностических записей задан корректный `axis_layer_id`.

## 7) Типовые ошибки и действия

- `axis_layer_id is required`
  - причина: для `iri`/`defects` не передана ссылка на ось;
  - действие: передать `axis_layer_id` существующего слоя `road_axis`.

- `Layer is not a road axis`
  - причина: в `axis_layer_id` передан слой другого типа;
  - действие: использовать слой `kind=road_axis`.

- `Unsupported axis format`
  - причина: неподдерживаемое расширение файла;
  - действие: использовать `.geojson`, `.csv` или `.zip` (Shapefile).

- `CSV must contain lon/lat columns`
  - причина: отсутствуют координатные поля;
  - действие: добавить `lon`/`lat` (или допустимые синонимы).

- `Shapefile ZIP must include .shp, .shx and .dbf`
  - причина: неполный архив shapefile;
  - действие: сформировать полный архив всех обязательных частей.

- `axis_layer_id must point to road_axis`
  - причина: в geolink передан не осевой слой;
  - действие: использовать слой с `kind=road_axis`.

- `Axis file missing` / `Failed to load axis`
  - причина: источник оси недоступен (удален файл или недоступен URL);
  - действие: восстановить источник оси и повторить расчет.
